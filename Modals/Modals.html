Unit: Modal Verbs 🛠️💡

1. The Core Rules of Modals

Before using modal verbs, keep these three structural rules in mind:

  • No "-s" for Singular: They never change forms.
    Incorrect: He cans code in HTML.
    Correct: He can code in HTML.
  • Followed by Base Verbs: Do not include "to" right after a modal.
    Incorrect: You should to study tonight.
    Correct: You should study tonight.
  • Simple Negatives: Just add "not" right to the verb.
    Incorrect: You don't must paste that code.
    Correct: You must not paste that code.

2. Categorizing Modals by Function

We choose modals based on the meaning or context we want to express:

A. Ability

• Can (Present): Celine can build beautiful websites on GitHub.
• Could (Past): Yesterday, I could not find where the loading glitch was.

B. Permission & Possibility

• May/Could (Polite Request): May I open a new issue on your repository?
• Might (Unsure Guess): It might rain later, so keep your iPad safe.

C. Advice & Suggestions

• Should: You should use a unique filename so GitHub doesn't give you an error.

D. Obligation & Necessity

• Must: You must commit your changes to save your code file.
• Must not (Forbidden): You must not share your GitHub password.

3. Quick Reference Summary Table
